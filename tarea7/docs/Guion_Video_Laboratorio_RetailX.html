TEXTO PARA DECIR:

BLOQUE 1 — Problema y objetivos (2 min)

TEXTO PARA DECIR:
RetailX tiene fuentes heterogéneas: CSV (ERP/POS), JSON (web/APIs), XML (IoT/GPS). Eso causa reportes lentos y KPIs distorsionados. Objetivos: diferenciar DW/Lake/Lakehouse, pipeline Medallion, Spark a escala y Delta con ACID.

TEXTO PARA DECIR:
DW es rígido y caro. Data Lake guarda crudo sin transacciones. Lakehouse + Delta Lake aporta ACID, versionado y time travel sobre almacenamiento barato.

  • delta/gold/.../_delta_log
